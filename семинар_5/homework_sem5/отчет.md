## 1. Новый инструмент `compare_periods`

```python
def compare_periods(
    metric: str,      # key_rate | fx_USD | fx_EUR | fx_CNY | cpi | unemployment
    period_a: str,    # YYYY-MM или YYYY-MM-DD
    period_b: str,
) -> dict:
    # → metric, a/b {date, value}, delta, ratio, source
```

Внутри вызывает `get_fx_rate`, `get_key_rate`, `get_inflation`, `get_unemployment` — без дублирования логики.

**Примеры запросов, где агент вызывает `compare_periods`:**

| Запрос | Вызов |
|---|---|
| «Во сколько раз вырос курс USD с января 2022 по апрель 2026?» | `compare_periods("fx_USD", "2022-01", "2026-04")` → ratio ≈ 1.09 |
| «На сколько п.п. изменилась ключевая ставка с июня 2022 по декабрь 2024?» | `compare_periods("key_rate", "2022-06", "2024-12")` → delta = 10 |
| «Как изменилась безработица с конца 2022 по конец 2024?» | `compare_periods("unemployment", "2022-12", "2024-12")` → delta = −1.4 |

Проверка из задания:

```bash
python agent.py "Во сколько раз вырос курс USD с января 2022 по апрель 2026?"
# → «примерно в 1.09 раза», compare_periods на шаге 1
```

### Что упало и почему (Q3)

Агент запросил `get_inflation(2026, 5)` — в CSV последний месяц **март 2026**, данных за май нет → `{"error": "нет данных ИПЦ на 2026-05"}`. Вместо отката на март 2026 агент **сразу выдал финальный текст** без `calculate`. Eval ждёт цепочку `get_key_rate + get_inflation + calculate`.

**Причина:** граничный случай «текущий месяц» + лаг публикации Росстата; в промпте нет явной инструкции брать последний доступный месяц при ошибке.

## 3. Диагностика — 3 разных типа ошибок

### Ошибка A — инструмент упал (отсутствие данных)

**Тип:** инструмент вернул `error`, агент не переиграл аргументы.

**Трасса** (`run_id: accce657`, Q3 eval):

```json
{"step": 1, "call": "get_inflation", "args": {"year": 2026, "month": 5},
 "obs": {"error": "нет данных ИПЦ на 2026-05"}}
{"step": 2, "final": "…точную реальную ставку сейчас вычислить нельзя…"}
```

**Причина:** CSV обрывается на `2026-03`; агент выбрал май по «текущей дате», не проверив доступность.

**Что исправить:** в промпте — «при ошибке get_inflation возьми последний месяц из ответа с source=rosstat_csv»; в `get_inflation` — опциональный режим `latest=True`.

---

### Ошибка B — исчерпан лимит шагов

**Тип:** бесконечный / незавершённый цикл обрезан `max_iter`.

**Трасса** (`run_id: ce9c370f`, `--max-iter 1`):

```json
{"step": 1, "call": "get_key_rate", "args": {"on_date": "2026-06-16"}, "obs": {"rate": 16.0, ...}}
{"step": 1, "call": "get_inflation", "args": {"year": 2026, "month": 5}, "obs": {"error": "..."}}
{"step": 1, "call": "get_unemployment", "args": {"year": 2026, "month": 5}, "obs": {"error": "..."}}
{"step": 1, "final": null, "error": "исчерпан лимит шагов max_iter=1"}
```

**Причина:** за один шаг агент параллельно запросил 3 инструмента, но не успел ни `calculate`, ни финал — лимит итераций.

**Что исправить:** для multi-hop вопросов `max_iter ≥ 4`; логировать предупреждение, если последний шаг — tool_calls без `final`.

---

### Ошибка C — неполная цепочка / преждевременный финал

**Тип:** неправильный порядок вызовов — пропущен обязательный `calculate`.

**Трасса** (тот же Q3):

```json
{"step": 1, "call": "get_key_rate", "obs": {"rate": 16.0, "source": "fallback_csv"}}
{"step": 1, "call": "get_inflation", "obs": {"error": "нет данных ИПЦ на 2026-05"}}
{"step": 2, "final": "…реальную ставку сейчас вычислить нельзя…"}
```

**Причина:** при частичном успехе модель считает вопрос «закрытым» объяснением, хотя eval и промпт требуют формулу `номинал − инфляция` через `calculate` (хотя бы с последним известным ИПЦ).

**Что исправить:** в system prompt — «если есть номинал и любой доступный ИПЦ — обязательно вызови calculate»; в eval для «сейчас» использовать якорную дату с полными данными.

## 4. Вывод: самый хрупкий элемент

**Самое слабое место — согласование «текущей даты» с лагом данных и обязательной цепочкой инструментов.**

Агент хорошо вызывает `compare_periods` и параллельные `get_*`, но при пограничных датах (нет свежего ИПЦ) ломается логика: ошибка инструмента → текстовый отказ вместо fallback на последний месяц + `calculate`. JSONL-трасса это хорошо ловит; без неё Q3 выглядел бы как «честный ответ», а не как сбой пайплайна.
