# Семинар 5 — Макро-агент ЦБ/Росстат (стартер)

ReAct-агент с пятью инструментами поверх живых сервисов ЦБ РФ и Росстата.
По ходу семинара заполняем TODO в `tools.py`, `schemas.py`, `agent.py` —
к концу занятия рабочий агент, отвечающий на макро-вопросы настоящими
числами от ЦБ, с прод-обвязкой (параллель, структура, кэш, бюджет).

## Установка

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Модель — через `.env` в корне семинара (см. `../.env.example`).

```bash
python -c "from llm_client import make_raw_client, get_model; print(get_model())"
```

## План семинара (база ~90 мин; со всеми блоками 5–10 — на 2–3 пары)

### Блок 0: разминка (0–15 мин)

Читаем `agent.py` построчно. Это тот же цикл, что на слайде 17 лекции 5:

- Системный промпт объясняет агенту, какие инструменты у него есть.
- В цикле модель получает `messages + tools` и возвращает либо финальный
  ответ, либо список `tool_calls` для исполнения.
- Мы исполняем инструменты, результаты добавляем обратно как `role: "tool"`.
- Цикл до `max_iter` или пока модель не скажет «ответ готов».

### Блок 1: запускаем и первый инструмент (15–40 мин)

1. Заполни три TODO в `agent.py` (разбор аргументов → вызов функции → ответ
   в контекст). Это поднимет цикл на ноги.
2. Реализуй `calculate` в `tools.py`. Проверь:

```bash
python agent.py "Сколько будет (21 - 9.5) делить на 2?"
```

Агент должен зайти в `calculate` один раз и выйти с числом 5.75.

3. Реализуй `get_key_rate`. Начни с запасного пути (CSV), затем — живой
   разбор страницы `https://www.cbr.ru/key-indicators/`:

```bash
python agent.py "Какая сейчас ключевая ставка?"
```

### Блок 2: остальные инструменты (40–60 мин)

Допиши `get_fx_rate` и `get_inflation`, добавь их JSON-схемы в `schemas.py`.

```bash
python agent.py "Сколько стоит доллар сегодня?"
python agent.py "Какая была инфляция в марте 2024?"
```

### Блок 3: полный прогон оценки (60–75 мин)

```bash
python eval.py
```

Прогонит 4 заготовленных вопроса и покажет, какие инструменты вызваны и
прошёл ли кейс:

```
[OK]   Q1 (2 шагов) — Какая сегодня ключевая ставка ЦБ?
[FAIL] Q3 (6 шагов) — Какая сейчас реальная ключевая ставка?
```

Если кейс упал — посмотри `eval_results.json`, открой трассу шагов и почини
системный промпт или описание инструментов.

### Блок 4: ломаем агента намеренно (75–85 мин)

Попробуй:

- `max_iter=1` → агент не успеет собрать цепочку и упадёт.
- Убери из системного промпта фразу «не придумывай числа» → агент начнёт
  отвечать своим обучающим знанием без вызовов инструментов. Посмотри трассу —
  часто финальная цифра случайная.
- Замени описание у `calculate` на «арифметика» → модель реже зовёт его,
  считает сама, ошибается.
- Удали одну валюту из CSV и сломай `_http_get` → `get_fx_rate` должен честно
  вернуть `{"error": ...}`, и агент должен это отработать.

### Блок 5: новый индикатор и тяжёлый multi-hop (85–110 мин)

Базовый агент справляется с простыми вопросами. Добавим пятый инструмент и
зададим вопросы, где нужно собрать 4–6 чисел из разных источников и посчитать.

1. Реализуй `get_unemployment(year, month)` в `tools.py` (источник —
   `data/unemployment_ru_monthly.csv`, логика 1-в-1 как у `get_inflation`) и
   добавь его JSON-схему в `schemas.py`.
2. Прогони тяжёлые вопросы — каждый разворачивается в цепочку вызовов:

```bash
python agent.py "Что сейчас выше: ключевая ставка или индекс нищеты (инфляция плюс безработица)?"
python agent.py "Какова реальная доходность годового вклада под ключевую ставку с поправкой на инфляцию?"
python agent.py "Сколько юаней за доллар по кросс-курсу ЦБ сегодня?"
```

Смотри трассу: агент сам раскладывает цепочку. На сложных вопросах он чаще
путает порядок и единицы — это и есть материал для блоков 6–10.

### Блоки 6–8: боевой агент (опц., если идёте быстрее)

Базовый цикл работает. Теперь три надстройки — всё в том же `agent.py`,
включаются флагами `--parallel`, `--structured`, `--critic` (или параметрами `run_agent`). По одному TODO на каждую:

- **Блок 6 — параллельные вызовы инструментов.** Модель за один шаг может
  попросить несколько инструментов сразу (например, ставку и инфляцию для
  реальной ставки). Исполняем их разом через `ThreadPoolExecutor`, а не по очереди.
- **Блок 7 — структурированный финальный ответ.** Вместо текста агент вызывает
  `submit_answer` и отдаёт Pydantic-схему `AgentAnswer` (ответ, число, единица,
  источники, уверенность). Тот же приём, что `RAGAnswer` в семинаре 4.
- **Блок 8 — самопроверка.** Перед тем как принять финал, отдельный вызов
  модели проверяет: выводится ли число из данных инструментов. Не выводится —
  возвращаем агента на доработку. Это про галлюцинации в агентных системах.

```bash
python agent.py --parallel "Сколько стоит доллар сегодня и сколько стоил 1 января 2022?"
python agent.py --parallel --structured --critic "Какая сейчас реальная ключевая ставка?"
```

На реальной ставке видно все три: ставку и инфляцию агент просит одним шагом
(параллельно), считает через `calculate`, отдаёт структуру, критик подтверждает.

### Блок 9: кэш инструментов (опц., прод-обвязка)

Инструменты детерминированы: `get_key_rate()` на сегодня всегда вернёт то же
число — незачем дёргать ЦБ дважды. Заполни два TODO «блок 9» в `_exec_one`:
ключ по имени и аргументам, проверка кэша до вызова, запись после. Проверь на
наборе вопросов — повторные вызовы видно сразу:

```bash
python eval.py --cache
```

В конце печатается, сколько обращений к инструментам пришло из кэша. Важно:
кэш экономит вызовы инструментов (сеть, латентность), но НЕ токены — за токены
отвечает кэш промпт-префикса на стороне провайдера (слайд 22 лекции).

### Блок 10: бюджет токенов и стоимость (опц., прод-обвязка)

Каждая итерация несёт в себе весь предыдущий контекст, поэтому стоимость растёт
нелинейно. Заполни TODO «блок 10» в `run_agent`: сними `resp.usage`
(`prompt_tokens`, `completion_tokens`), посчитай стоимость, добавь в `usage_log`.

```bash
python agent.py --cost "Что сейчас выше: ключевая ставка или индекс нищеты?"
```

Печатается таблица «шаг → токены → $/шаг → $ накопительно» — тот самый
квадратичный рост со слайда 22.


## Справка по сервисам ЦБ

- Курсы валют на дату:
  `GET https://www.cbr.ru/scripts/XML_daily.asp?date_req=22/04/2026`
  Ответ — XML (windows-1251), теги `<Valute>` → `<CharCode>`, `<Value>`, `<Nominal>`.
- Ключевая ставка сегодня:
  `GET https://www.cbr.ru/key-indicators/` (HTML, число со знаком %).
- История ключевой ставки: `data/key_rate_history.csv` (по дате).

## Когда застрял

- Модель не зовёт инструменты → проверь, что `tool_choice="auto"` (а не `"none"`)
  и что `tools=TOOL_SCHEMAS` передаются в запрос.
- `role: "tool"` без `tool_call_id` → модель видит вызов без пары и путается;
  id должен совпадать с тем, что пришёл от неё в `tool_call.id`.
- Агент бесконечно зовёт один и тот же инструмент → он не видит результат,
  потому что ответ инструмента не добавили в `messages`. Проверь TODO-3 в `agent.py`.
