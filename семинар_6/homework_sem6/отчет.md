
Встроен в `run_pwc()` сразу после `planner()`: при ошибках — повторный вызов
`planner(question, feedback=f"Инструменты не существуют: {errors}")`.

**Пойманные типы выдуманных инструментов:**

| Тип | Пример в плане | Ошибка валидатора |

| Инструмент из другого ДЗ | `compare_periods` | `неизвестный инструмент 'compare_periods'` |

```bash

```

Без валидатора Планировщик может добавить `get_cumulative_inflation`. С валидатором — перепланирование: либо честный план из `get_inflation` × N + `calculate`, либо `subquestions=[]` с объяснением.

## 5. Вывод: когда PWC оправдан

**PWC оправдан**, когда:
- нужна декомпозиция с зависимостями (`depends_on`) и параллель на уровне;
- Планировщик склонен к выдуманным tools — **валидатор схемы обязателен**;
- задача multi-hop (реальная ставка, кросс-курс) — выигрыш над одиночным агентом.

**Одиночный агент лучше**, когда:
- вопрос на 1–2 tool calls (простой курс, одна ставка);
- overhead PWC (3+ LLM-вызова: план, N workers, критик, синтез) не окупается;
- накопленная инфляция за 50+ месяцев — даже честный план слишком тяжёл для исполнителя.

Самый хрупкий элемент — **согласованность Планировщика и валидатора с возможностями Исполнителя** (не выдумывать tools и не планировать 51 вызов без декомпозиции на батчи).
