# Семинар 6 — Планировщик-Исполнитель-Критик (стартер)

Мультиагент поверх одиночного агента из С5: Планировщик декомпозирует
вопрос, Исполнитель отвечает на каждый подвопрос (тот же С5-агент как
функция), Критик проверяет согласованность, оркестратор крутит цикл.

По ходу семинара закрываем 8 TODO в четырёх модулях — к концу занятия
у тебя рабочий PWC, проходящий 2 из 3 тестов, где одиночный агент
ломается.

```bash
python agent_s5.py "Во сколько раз USD подорожал с 1 января 2022 по сегодня?"
```

### Блок 1 — Планировщик

Закрой 2 TODO в `planner.py`:
1. Добавь в SYSTEM_PROMPT правило про обязательный calculate-подвопрос.
2. Реализуй параметр обратной связи для перепланировки.

Проверка:
```bash
python planner.py "Во сколько раз USD подорожал с 2022?"
```
Должен вывести 3 подвопроса, последний — с `calculate`.

### Блок 2 — Исполнитель

Закрой 1 TODO в `worker.py`: собери `prev_context` из `depends_on`.

Проверка: запусти код из `if __name__ == "__main__"` блока
в worker.py — должен ответить курсом USD за 1-2 шага.

### Блок 3 — Критик + Маршрутизатор

Закрой 2 TODO в `critic.py`:
1. Передай в промпт ТОЛЬКО `answer + used_tools`, НЕ `raw_trace`.
2. Поставь `temperature=0.7`.

Закрой 3 TODO в `orchestrator.py`:
1. `_topological_sort` — DFS по `depends_on`.
2. Ветки `replan` / `rework` в основном цикле.
3. `_synthesize` — одной LLM-call собери финальный ответ.

Интеграционный прогон:
```bash
python orchestrator.py "Во сколько раз USD подорожал с 1 января 2022 по сегодня?"
```

### Блок 4 — Eval

```bash
python eval_pwc.py --single
```

Прогонит 3 кейса на одиночном агенте и на PWC, напечатает сравнение.
Если Q3 падает — это ПРАВИЛЬНО. Паттерн не чинит галлюцинации в плане.

### прод-паттерны

**Блок 6 — Диспетчер.** `supervisor.py`: вместо статичного плана агент-диспетчер
решает следующий шаг по ходу (слайд 13 Л6). Закрой 2 TODO: правило про `calculate`
в `SUPERVISOR_SYSTEM` и ветку `ask`/`finish` в `run_supervisor`.

```bash
python supervisor.py "Во сколько раз USD подорожал с 1 января 2022 по сегодня?"
```

Сравни с PWC: диспетчер гибче, но менее предсказуем и может ходить лишние шаги.

**Блок 7 — «Ультиматум»** `ultimatum.py`: два агента (Предлагающий/Отвечающий)
играют в «Ультиматум» (блок 3 Л6). Закрой 2 TODO — оживи обоих игроков через
`make_client`.

```bash
python ultimatum.py 10
```

Ожидаемо: низкие предложения Отвечающий отвергает, ближе к 50/50 — принимает. Это
эталон «LLM против LLM».
